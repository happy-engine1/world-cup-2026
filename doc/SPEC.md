# ワールドカップ2026 対戦トーナメント表 — 詳細仕様

> 非公式ファンサイト。FIFAワールドカップ2026（北中米大会・48カ国/12グループ）の
> グループ順位・3位通過・決勝トーナメントを1画面で閲覧できる。
> 最終更新時点: グループステージ完了（2026-06-28）。

---

## 1. 概要

| 項目         | 内容                                                          |
| ------------ | ------------------------------------------------------------- |
| 目的         | 大会の順位とトーナメント対戦表を見やすく可視化（予測機能つき）   |
| 種別         | 非公式・ファンメイド（FIFA・大会主催者とは無関係）            |
| リポジトリ   | https://github.com/happy-engine1/world-cup-2026               |

## 3. 画面レイアウト

### デスクトップ（xl = 1280px 以上）
- ヘッダー（1行）: 左=サブタイトル＋ドラッグヒント / 中央=タイトル＋「非公式・ファンメイド」バッジ / 右=選択中メッセージ＋言語切替
- 本体3カラム: 左 = グループ A–F / 中央 = 決勝トーナメント / 右 = グループ G–L
- 3位チーム順位表: トーナメントの下（中央カラム内）
- フッター: 非公式ディスクレーマー

### モバイル/タブレット（xl 未満）
- ヘッダーは中央寄せで縦積み（折り返し）
- 本体は1カラム縦積み。**並び順は ①トーナメント → ②A–F → ③G–L**（CSS order で制御）
- 3位チーム順位表は最下部（A–L の下）
- 余白を縮小し全幅利用。縦・横どちらでもページ横スクロールは発生しない
  （トーナメント図のみ枠内で横スクロール）

## 4. データモデル（`src/data/groups.ts`）

- 48チーム × 12グループ。出典は FIFA 公式順位表のスナップショット（手動取得）。
- 1チームの保持項目: `code`(flagcdnコード) / `nameJa` / `played` / `win` / `draw` / `loss` / `gf` / `ga` / `tcs` / `pts`
- 多言語名は `TEAM_NAMES_INTL`（en/zh/ko）＋ `teamName(code, lang)` で解決。
- `TCS` = Team Conduct Score（フェアプレーポイント、FIFA公式値。0が最良・負ほど悪い）。
- `FINAL_GROUPS`: 全3試合を終え順位が確定したグループの集合（バッジ「確定/途中」に対応）。
- 順位ソート `rankGroup(g)`: **勝点 → 得失点差 → 総得点 → TCS → 組記号**
  （別グループ比較のため当該チーム間の直接対決ステップは省略）。
- 3位ランキング `rankThirdsBy(orderOf)`: 各組3位を上記基準で並べる（`rankedThirds()` は実データ版）。

---

## 5. グループ順位表（`src/components/GroupTable.tsx`）

- 折りたたみ時カラム: **順位 / 国旗 / 国名 / 勝点**（成績順ソート済み）
- 見出しクリックで展開（高さアニメ）→ **試合 / 勝 / 分 / 敗 / 得点 / 失点 / 得失 / TCS / 勝点**
- バッジ: 確定（緑）/ 途中（青）。順位帯色: 1–2位=金 / 3位=シアン / 4位=灰
- 行クリックでそのチームを選択 → トーナメントの対戦ルート＆対戦候補をハイライト連動
- 選択行は薄い黄色（amber）で強調（トーナメントと統一）

---

## 6. 3位チーム順位表（`src/components/ThirdPlaceTable.tsx`）

- 全12グループの3位を成績順に一覧。**上位8チームがベスト32へ進出**（緑強調＋8/9位の間に区切り線、9位以下は淡色）
- カラム: 順位 / 組 / 国旗 / 国名 / 試合 / 勝 / 分 / 敗 / 得点 / 失点 / 得失 / TCS / 勝点
- グループ内順位を変更すると、その組の3位チーム表示も自動で連動。

---

## 7. 決勝トーナメント（`src/components/Bracket.tsx`）

- 32チーム: **ベスト32(1/16) → ベスト16(1/8) → 準々決勝(1/4) → 準決勝(1/2) → 決勝**＋3位決定戦
- シード配置は公式組み合わせ（`w-tmp/fifa/image-2.png` 準拠）。左右の山で準決勝へ集約。
- 各カラム上部にラウンド名（1/16 等）。中央に自作SVGトロフィー＋決勝枠。
- 各ベスト32枠の表示:
  - 出場元ラベル（例: `C1` / `F2` / `C/E/F/H/I3`）
  - 国旗＋国名（確定=濃色、未確定=薄字イタリック＋「暫定」バッジ）
  - 試合日時（言語のタイムゾーンに追従。§10）
- 進級ライン（エルボー型の接続線）で各試合→次ラウンドを接続。
- チーム選択時、勝ち上がりルートと対戦候補を金/シアンでハイライト。

## 8. 3位通過枠の割当ロジック（FIFA Annex C 準拠）

2026方式では各3位枠が「特定5組のいずれかの3位」だけを受け入れる。どの8組の3位が
通過したか（全495通り）に応じて、**FIFA公式 大会規定 Annex C の対応表**で配置が一意に決まる。

実装（`src/data/bracket.ts`）:
- `LEAVES`: 32枠の並び。3位枠は許容グループ集合 `allowed` を持つ。
- `OFFICIAL_THIRD_ASSIGNMENT`: 確定済み組合せ → 公式どおりの割当（リーフindex→グループ）。
- `computeSeeds(orderOf, qualifyingThirds)`:
  - 通過8組の組合せキーが `OFFICIAL_THIRD_ASSIGNMENT` にあれば**公式表どおり**に配置。
  - 無ければ `assignThirds()`（許容集合を満たす二部マッチング＝近似）にフォールバック。
- 現行の最終確定組合せ **{B,D,E,F,I,J,K,L}** を登録済み → ベスト32全16試合がFIFA公式と一致。
- 全グループ確定かつ公式割当使用時は、3位通過枠も `confirmed`（暫定バッジを外す）。

> 予測モードでユーザーが別の8組を通過させた場合は、その組合せが未登録なら近似配置になる
> （FIFA公式と一部異なる可能性あり）。実データの組合せは公式一致。

## 9. 多言語対応（`src/i18n/`）

- 対応言語: 日本語 `ja` / 英語 `en` / 中国語 `zh` / 韓国語 `ko`。**既定は中国語**。
- `I18nProvider`: 言語stateを保持。`localStorage`（キー `wc2026-lang`）に保存・復元。`<html lang>` も同期。
- `ui.ts`: 全UI文字列の辞書＋出場元ラベル生成 `seedLabel()`。
- 翻訳対象: ヘッダー/凡例/操作ヒント/グループ表カラム/ラウンド名/3位表/フッター/48カ国名 等すべて。
- サブタイトル（大会名）は各言語の通称: 日本語=ワールドカップ / 英語=Canada · Mexico · USA World Cup / 中国語=美加墨世界杯 / 韓国語=월드컵。

## 10. 試合時刻のタイムゾーン

`MATCH_DATES` は**日本時間（JST=UTC+9）基準**で保持し、言語ごとに時差表示する。

| 言語   | タイムゾーン        | JSTからの差 |
| ------ | ------------------- | ----------- |
| 日本語 | 日本時間 (JST)      | ±0          |
| 한국어 | 한국 시간 (KST)     | ±0（同値）  |
| 中文   | 中国时间 (CST)      | −1時間      |
| English | US Eastern (ET)    | −13時間     |

- `matchTime(key, lang)` が時差計算（日付繰り上げ/繰り下げ対応）。
- 凡例下に現在のタイムゾーン（`tzLabel`）を表示。
- 「アメリカ時間」は会場分散のため代表として**米国東部時間 (ET)** を採用。

## 11. 予測（ドラッグ並べ替え）機能

「未来の結果を予測 → トーナメントの対戦相手を探る」what-ifツール。

- **グループ順位**（未確定グループのみ）と**3位チーム順位表**（全12）を、左端のハンドル(⠿)で
  ドラッグ並べ替え可能（Framer Motion `Reorder`、PC/スマホ対応・縦スクロールと非干渉）。
- 並べ替えると `computeSeeds` / `rankThirdsBy` が再計算され、**ベスト32の対戦カード・3位通過枠が即連動**。
- グループ内順位を変えると3位表の該当チームも連動。
- 並べ替え中は「予測モード」バナーを表示し、**リセットボタン**で実データに復帰。
- 状態管理は `src/app/page.tsx`（`predictions` / `thirdOrder`）。
- 注: 予測は「並び順」のみ（勝点等は実値のまま再計算しない）。

## 14. デプロイ（さくらVPS / Ubuntu）

公開リポジトリを VPS 上で `git pull` →ビルド→ nginx が `out/` を配信（鍵/Secretsを預けない）。

```bash
cd ~/world-cup-2026
./deploy.sh        # git reset --hard origin/main → npm ci → npm run build（out/生成）
```

- nginx は `out/` を `root` に向ける（`deploy/nginx.conf.sample`）。HTTPSは certbot。
- 詳細は `DEPLOY.md`。メモリ1GBはビルド時スワップ推奨。

## 15. データ更新の運用フロー

1. FIFA公式 standings をブラウザ（Playwright等）で取得（JS描画のため静的取得不可）。
2. `src/data/groups.ts` の該当グループ数値を更新。3試合終了したグループは `FINAL_GROUPS` に追加。
3. 3位通過8組の組合せが変わったら、FIFA公式ブラケットの3位配置を確認して
   `bracket.ts` の `OFFICIAL_THIRD_ASSIGNMENT` に新キーを追加（公式一致を維持）。
4. `npm run build` で型/ビルド確認 → commit & push。
5. VPS で `./deploy.sh`。

---

## 16. 既知の制約・前提

- データは取得時点のスナップショット（自動更新なし）。
- 予測モードの3位割当はFIFA公式表に未登録の組合せでは近似。
- 「アメリカ時間」はET固定（会場ローカル時刻ではない）。
- TCSは公開値のみ反映（FIFAが同点判定時に公表する値）。
